## CarCue iOS App — Afternoon Dev Session (~4:00-5:15 PM)

### Features Built/Fixed
1. **ExportService build fix** — `yearString` not visible in ExportService scope, changed to `String(vehicle.year)`

4. **Edit Service Records** — New `EditServiceView.swift`. Tap any service row → edit sheet with full form (types, date, cost, shop, notes, photos) + delete button. Context menu also has Edit option.
5. **Service history refresh bug** — After editing a service, the list/summary didn't update. Fixed with `refreshID = UUID()` pattern on sheet dismiss.
6. **Onboarding flow** — 5 swipeable pages (Garage, Service History, Reminders, Fuel, Recall Alerts). Themed accent colors per page, animated dots, Skip/Next/Get Started. Shows once (UserDefaults `hasCompletedOnboarding`).
7. **Delete vehicle** — Long-press context menu on garage cards → confirmation dialog warning about cascade delete of all records.
8. **Service filter chips fix** — Split concatenated "Oil Change + Air Filter + Cabin Filter" into individual pills. Filter logic uses `.contains()` on split array.
9. **VIN auto-correction** — `sanitizeVIN()` replaces I→1, O→0, Q→0 before NHTSA decode (these letters are banned from VINs). Justin hit this exact issue with his Jeep Wrangler VIN.
10. **OCR receipt scanner rewrite** — Major overhaul:
    - OCR error correction dictionary (50+ common Vision misreads for auto terms)
    - Fuzzy matching with Levenshtein distance (40% tolerance for handwriting)
    - `containsKeywordLoose()` — strips spaces, sliding-window substring matching
    - Multi-service detection (finds ALL types, not just first)
    - Smart shop name extraction (skips service type lines)
    - Line item descriptions cleaned to canonical service type names
    - Total = sum of line items when multiple found
    - Debug raw OCR text view (`#if DEBUG`)
