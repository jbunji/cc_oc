## SHIELD.md Implementation

### First attempt (incorrect)
- Built custom format without finding the official spec
- PDF extraction failed

### Second attempt (correct)
Justin shared the real spec: https://nova-hunting.github.io/shield.md/

Rewrote SHIELD.md following official Nova-Hunting format:
- **Context-based runtime threat feed policy** (v0 = guidance, v1 = hard enforcement)
- **DECISION block** must be output before any skill install/execute, tool call, MCP, network, secrets
- **Threat entries** with: id, fingerprint, category, severity, confidence, action, recommendation_agent, expires_at, revoked
- **10 local threats** (LOCAL-001 through LOCAL-010) covering our learned lessons
- **Confidence threshold**: >= 0.85 for enforceable
- **Hard stop rule**: If action = block, respond with threat match and stop immediately
- Can integrate MoltThreats feed for known external threats

Key differences from my first attempt:
- Official format uses structured threat entries, not just rules
- Requires DECISION block output before actions
- Has recommendation_agent mini-syntax (BLOCK/APPROVE/LOG with conditions)
- Context limits: cap 25 threats loaded
- Forward-compatible with v1 runtime enforcement

### Final Decision
Justin said "do what you think" — balance safety without overkill. Slimmed SHIELD.md from 8KB ceremony to 1.4KB quick reference table:
- 10 threat entries in a simple table (block vs ask)
- Red flags list
- Trust tiers
- Link to full spec if we ever need it

Primary security guidance stays in AGENTS.md. SHIELD.md is now just a lean reference doc, not an active policy I consult on every action. Ready for MoltThreats integration later if wanted.
