## 8:40 AM - 11:09 AM — Ledgr Build & Debug Session (Continued)

### Testing & Bug Fixes (iterative with Justin)

**Justin tested on real iPhone — found multiple issues, fixed all:**

1. **Contrast issues (light mode):** `.toolbarColorScheme(.dark)` was forcing light nav title text — invisible on white. Removed from ALL 9 views. Bumped text colors darker.

2. **PilotBar didn't do anything:** Was just `print()`. Fixed two ways:
   - **Inline mode:** Created `AppState` shared across tabs. `sendInline()` processes agent request WITHOUT switching tabs. Data refreshes via `@Query`. Shows green toast "✅ Logged $99..." below PilotBar.
   - **Smart tab switch:** If agent asks follow-up question (needs more info like "where did you spend?"), auto-switches to Pilot tab. If action completed, stays on current page.

3. **Double expense creation (PERSISTENT BUG — 3 iterations to fix):**
   - Root cause: Gemini 2.0 Flash model calls `create_expense` TWICE in one response for a single expense
   - Fix 1: Updated server prompt "ONE EXPENSE PER STATEMENT" — helped but not reliable
   - Fix 2: Server-side dedup on tool_calls — strips duplicate `create_expense` with same amount+merchant before sending to client
   - Fix 3: Client-side dedup — same amount+merchant within 10 seconds gets silently accepted (returns success to model so it doesn't ask for confirmation)
   - Fix 4: Re-enabled `[EXPENSE_LOGGED]` tag handler with dedup guard — model sometimes returns text-only (no tool calls), need tags as fallback

4. **Raw `[EXPENSE_LOGGED]` tags visible in chat:** Parser wasn't catching inline tags. Added regex cleanup pass in `AgentResponseParser` to strip any remaining tags from display text.

5. **"Can you confirm?" confusion:** Dedup was returning "duplicate skipped" to model → model asked for confirmation. Changed to return SUCCESS message so model thinks it worked (it already did).

6. **Tab switch too aggressive:** Question mark in "Can you confirm?" triggered Pilot switch. Narrowed detection to only MISSING INFO patterns: "where did", "what did", "how much", etc. Also checks if action was already completed — if so, never switches.

7. **Onboarding "How do you spend your money?" removed:** Replaced old 4-step (welcome → budget → category picker → allocation) with new 4-step (welcome → income → bills → done). Categories auto-seed. Bills page has quick-add chips + custom "Add a bill" field with auto-focus.

8. **Custom icons wired into UI:** `CategoryProgressRow` now shows generated indigo line art from asset catalog instead of colored dots. Gold accent on "FREE TO SPEND TODAY" label. SF Rounded typography on section headers.

### Lessons Learned
- **Gemini 2.0 Flash is unreliable with tool calls** — frequently calls the same tool twice for a single user intent. Must dedup on BOTH server and client.
- **Never return error/skip messages to the model** — it interprets them as failures and asks for confirmation. Always return success.
- **Tag-based AND tool-based creation both needed** — the model inconsistently uses one or the other. Need both paths with dedup guard.
- **`.toolbarColorScheme(.dark)` is a dark-mode-only modifier** — breaks light mode nav titles. Never use it in apps that support both modes.
