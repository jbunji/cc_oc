### Share Trip Zero-Byte File Bug
- **Problem:** "Share Trip" → Save to Files created 0-byte file, blank white page when opened
- **Root cause:** Passing raw String to ShareSheet, iOS doesn't handle "Save to Files" for raw strings
- **Fix:** Write to temp file first (`TripStash-{trip-name}.txt`), share the file URL instead
- **Result:** Proper .txt files with full itinerary content

### CloudKit Migration (CRITICAL - 5+ hour agent task)
Mandatory for production. CloudKit was disabled because it caused silent store failures.

**What changed:**
- **All 5 SwiftData models** (Trip, TripEvent, PackingItem, TripDocument, BudgetItem) refactored:
  - Every `@Model` property now has a default value (UUID() → UUID() = UUID(), String → String = "", etc.)
  - All relationships changed from `[Type] = []` to `[Type]? = []` (CloudKit requirement)
- **50+ references updated** across 19 view/service/viewmodel files:
  - `trip.events` → `(trip.events ?? [])`
  - `.append()` operations wrapped in nil checks
  - `.count`, `.isEmpty`, `.sorted`, `.filter` all need `?? []`
- **TripStashApp.swift:** `cloudKitDatabase: .none` → `.automatic`
- **Entitlements:** Already had correct iCloud/CloudKit config

**Impact:**
- Data now syncs to iCloud automatically
- Survives app reinstalls, device changes
- Schema migrations won't nuke data anymore
- ⚠️ First run after this update will start fresh (old local DB vs new CloudKit schema incompatible)
- Requires `iCloud.com.bunjiapps.tripstash` container registered in Developer portal

### Translation JSON Parsing Fix
- **Problem:** Translation results showing raw JSON instead of translated text
- **Cause:** Gemini wrapping JSON in extra text ("Here's the translation:\n```json\n{...}\n```")
- **Fix:** Extract just the `{...}` portion, parse `translation` + `pronunciation` fields
- **Fallback:** Show raw response if parsing fails (better than crashing)
