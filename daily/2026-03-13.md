## 11:30 AM — Widget Fix Saga
- Widget existed but wasn't working — multiple issues discovered and fixed:
  1. **Widget not showing in search**: Widget target wasn't embedded in main app. Added `dependencies: - target: HomeKeepWidget` to project.yml
  2. **Bundle ID prefix error**: Widget's manual Info.plist was missing CFBundleIdentifier. Switched to `GENERATE_INFOPLIST_FILE: YES`
  3. **CFBundleVersion null**: Widget missing MARKETING_VERSION and CURRENT_PROJECT_VERSION build settings. Added to match app (1.0.0 / 1)
  4. **Widget showing placeholder data, not real tasks**: Rewrote widget architecture — ditched SwiftData in widget (can't share across targets easily). Now uses App Group UserDefaults:
     - `WidgetDataService.swift` — writes top 5 tasks + score to shared UserDefaults
     - Widget reads from UserDefaults (no SwiftData dependency)
  5. **"container_create_or_lookup_app_group_path_by_app_group_identifier: client is not entitled"**: Main app target didn't have App Groups capability added in Xcode — only widget had it. Justin added it manually in Xcode Signing & Capabilities
  6. **All data disappeared**: Had moved SwiftData store to App Group container, which created a new empty DB. Reverted to default location — App Group only used for UserDefaults widget pipe
- Widget now updates on: app launch (MainTabView.onAppear), task completion, app backgrounding
- Small widget: shows most urgent task + score %
- Medium widget: shows top 3 tasks + overdue count
